# Заявки з сайту → Google Таблиця

Покрокова інструкція, як зробити так, щоб заявки з форми на сайті потрапляли у твою Google Таблицю.

## 1. Створи таблицю
1. Зайди на [sheets.google.com](https://sheets.google.com) → **Пусту таблицю**.
2. Назви її, наприклад, «Заявки — Господар».

## 2. Додай скрипт
1. У таблиці: **Розширення → Apps Script**.
2. Видали весь код-заглушку, який там є.
3. Відкрий файл `Code.gs` з цієї папки (`google-apps-script/Code.gs`), скопіюй увесь вміст і встав у редактор Apps Script.
4. Збережи (значок дискети або Ctrl+S). Назви проєкт, наприклад, «Gospodar Forms».

## 3. Опублікуй як Web App
1. Угорі праворуч — **Deploy → New deployment**.
2. Натисни на значок шестерні біля "Select type" → вибери **Web app**.
3. Заповни:
   - **Description**: будь-який текст, наприклад "Форма заявок"
   - **Execute as**: **Me** (твій акаунт)
   - **Who has access**: **Anyone** (обов'язково "Anyone", інакше сайт не зможе надіслати дані)
4. Натисни **Deploy**.
5. Google попросить надати дозволи — підтверди (Advanced → Go to [назва проєкту] → Allow), це нормально, скрипт твій власний.
6. Скопіюй **Web app URL** — він виглядає так:
   `https://script.google.com/macros/s/AKfycb............/exec`

## 4. Встав URL у проєкт сайту
1. Відкрий файл `src/config.js`.
2. Встав скопійований URL:
   ```js
   export const GOOGLE_SHEETS_ENDPOINT = 'https://script.google.com/macros/s/AKfycb.../exec'
   ```
3. Перезбери сайт (`npm run build`) і задеплой (`npm run deploy`).

## 5. Перевір
Заповни форму на сайті й надішли — у Google Таблиці має з'явитися новий рядок з датою, ім'ям, телефоном, напрямком послуги та коментарем.

## Якщо потрібно оновити скрипт пізніше
Якщо зміниш код у `Code.gs` (наприклад, додаси нову колонку), потрібно зробити **New deployment** ще раз (Deploy → Manage deployments → значок олівця → New version → Deploy) — інакше зміни не застосуються до вже опублікованого Web App.

## Примітка про надійність
Форма надсилає запит у режимі `no-cors` — це стандартний спосіб роботи з Apps Script Web App з браузера. Через це сайт не може прочитати відповідь сервера напряму, тож повідомлення "Заявку прийнято" зʼявляється одразу після успішної відправки запиту (без очікування підтвердження від таблиці). ЯкщоURL у `config.js` порожній або невірний — форма покаже повідомлення про помилку.
