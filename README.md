# Господар Кам'янця-Подільського — лендінг

Багатосторінковий сайт послуг: сад та ландшафт, скупка металобрухту й радіодеталей,
демонтажні роботи, вивіз сміття та продаж дров. React + Vite + CSS Modules, react-router-dom.

## Структура сайту
- `/` — головна (hero, 4 картки напрямків, як відбувається робота, чому саме я, CTA)
- `/poslugy` — огляд усіх 4 напрямків
- `/poslugy/sad-ta-landshaft` — сад та ландшафт (11 послуг + сезонний календар)
- `/poslugy/skupka` — скупка металобрухту та радіодеталей (6 послуг)
- `/poslugy/demontazh` — демонтажні роботи (4 послуги)
- `/poslugy/vyviz-ta-drova` — вивіз сміття та продаж дров (6 послуг)
- `/kontakty` — телефони, мапа, форма заявки

Усі послуги описані в одному місці — `src/data/categories.js`. Щоб додати чи змінити
послугу, редагуй масив `services` відповідної категорії там.

Результат — у папці `dist/`.

## Деплой на GitHub Pages

1. Якщо репозиторій НЕ `username.github.io`, а звичайний (наприклад `Hospodar`), відкрий
   `vite.config.js` і встанови `base` — **точно як назва репозиторію, з тим самим регістром**:
   ```js
   export default defineConfig({
     plugins: [react()],
     base: '/Hospodar/',
   })
   ```
   Якщо репозиторій саме `elinamrachkovska.github.io` (кореневий), `base` не потрібен.

2. Встанови пакет для деплою:
   ```bash
   npm install --save-dev gh-pages
   ```

3. Додай у `package.json` у розділ `"scripts"`:
   ```json
   "predeploy": "npm run build",
   "deploy": "gh-pages -d dist"
   ```

4. Задеплой:
   ```bash
   npm run deploy
   ```

5. У налаштуваннях репозиторію на GitHub → Pages → Source постав гілку `gh-pages`.

Сайт коректно працює і при прямому переході на внутрішні сторінки (наприклад,
`/Hospodar/poslugy/skupka`) завдяки `public/404.html` — стандартний обхід того, що
GitHub Pages не підтримує server-side rewrite для SPA.

## Медіа на сторінках категорій
Реальні фото та відео вже додані:
- **Сад та ландшафт** — 10 фото (обрізка, зняття дерев, формування туй/самшиту, ландшафтний дизайн) + 2 відео, у `public/media/sad/` та `public/media/videos/`.
- **Скупка** — 6 фото плат, роз'ємів, транзисторів, у `public/media/skupka/`.
- **Вивіз сміття та дрова** — 1 реальне фото (колоди/пеньок) у `public/media/vyviz/` + 2 ліцензійні стокові фото з Unsplash (безкоштовні для комерційного використання).
- **Демонтаж** — реальних фото поки нема, використано 3 ліцензійні стокові фото з Unsplash. Заміни на реальні кадри, коли зʼявляться — просто онови масив `media.photos` відповідної категорії в `src/data/categories.js`.

Щоб додати чи замінити фото/відео: онови масив `media.photos` / `media.videos` у `src/data/categories.js`. Локальні файли кладуться в `public/media/<категорія>/`, шлях у масиві вказується відносно `public/` (без початкового `/`).

## Соцмережі
Instagram, Facebook, TikTok підключені в `src/data/categories.js` (`contact.social`) — відображаються у футері та на сторінці контактів.

## Заявки з форми → Google Таблиця
Форма на `/kontakty` вміє надсилати заявки у Google Таблицю. Покрокова інструкція —
у `google-apps-script/README.md`. Коротко:
1. Створи Google Таблицю, встав код з `google-apps-script/Code.gs` у Apps Script.
2. Опублікуй як Web App ("Anyone" має доступ).
3. Встав отриманий URL у `src/config.js` → `GOOGLE_SHEETS_ENDPOINT`.
4. Перезбери й задеплой сайт.

Поки `GOOGLE_SHEETS_ENDPOINT` порожній, форма просто показує "Заявку прийнято" без
реальної відправки — це навмисно, щоб сайт не ламався до налаштування таблиці.

## Що потрібно донастроїти
- **Фото для "Демонтаж"** — поки що стокові фото з Unsplash (ліцензійно чисті, безкоштовні для комерційного використання), бо реальних кадрів цього напрямку ще нема. Заміни на свої, коли з'являться — масив `media.photos` категорії `demontazh` у `src/data/categories.js`.
- **Ціни на послуги** — якщо з'являться, додай поле `price` в `src/data/categories.js`
  і виведи його в `ServiceCard`.
- Перевір регіон обслуговування ("Кам'янець-Подільський та область") у `src/data/categories.js` — вказано за замовчуванням.
- Логотип — монограма "ГКП" у медальйоні (`public/logo.png` + фавікони). Вихідний SVG —
  `logo-source.svg` у корені проєкту, якщо захочеш підправити кольори чи текст.
